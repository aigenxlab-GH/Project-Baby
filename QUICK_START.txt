STEP 4: SPOT CHECK - ~10 minutes
─────────────────────────────────

Pick 4-5 files to verify manually:

Check a PRODUCT REVIEW file:
  - Should have 💡 ABOUT THESE REVIEWS after first ## heading
  - Should have 📋 SOURCES & FACT-CHECKING before Related Articles
  - Example: content\products\baby-bathtubs\angelcare-soft-touch-bath-support-review.mdx

Check a HEALTH BLOG file:
  - Should have ⚠️ MEDICAL DISCLAIMER after first ## heading
  - Should have 📋 SOURCES & FACT-CHECKING before Related Articles
  - Example: content\blog\gestational-diabetes-pregnancy.mdx

Check a PARENTING GUIDE file:
  - Should have 📋 SOURCES & FACT-CHECKING before Related Articles (NO medical/product disclaimer)
  - Example: content\parenting\newborn\umbilical-cord-care-keeping-it-clean-and-dry.mdx

Check a GENERAL BLOG file:
  - Should have 📋 SOURCES & FACT-CHECKING before Related Articles (NO medical/product disclaimer)
  - Example: content\blog\baby-name-generator-guide.mdx

How to spot check:
  1. Open file in VS Code
  2. Search for "MEDICAL DISCLAIMER" or "ABOUT THESE REVIEWS" or "SOURCES & FACT-CHECKING"
  3. Verify it appears AFTER first H2 heading (for medical/product)
  4. Verify sources appears BEFORE "## Related Articles"

STEP 6: VERIFY DEPLOYMENT - ~5 minutes

4. If all looks good, you're done!

WHAT IF SOMETHING GOES WRONG?

Problem: Script shows errors
Solution:
  - Check you're in the right directory
  - Verify Python is installed: python --version
  - Check FILE_INVENTORY.txt for expected file structure

Problem: Dry run shows wrong categorization
Solution:

Solution:

  - Identify the specific issue
  - Update script if needed
  - Re-run

Problem: Something else?
Solution:
  - git checkout -- content/  (revert all changes)
  - Review EXECUTION_SUMMARY.md for troubleshooting
  - Check specific modified file manually
  - Diagnose root cause
